# Repo Guidance

- Never include local machine paths in GitHub-facing files.
- Never include personal names, company names, client names, or internal identifiers in GitHub-facing files unless they are intentionally public project metadata.
- Prefer relative paths and neutral identifiers in docs, source comments, release notes, and other repository content.
- Do not commit build outputs from `bin/` or `obj/`. Binaries will be published through GitHub Releases instead.
